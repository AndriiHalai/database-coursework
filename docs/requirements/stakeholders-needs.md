## Функціональність

Всі функції нашої системи управління проєктами втілені для різних категорій користувачів і працівників у вигляді інтерфейсів. Завдяки цьому можна забезпечити повне та наглядне виконання вимог, поставлених системі, визначити її межі, продемонструвати можливості, доступні різним типам учасників, та взаємозв’язки між ними.

### Інтерфейс звичайного користувача

Цей інтерфейс представляє собою мінімально достатній для участі у розробці проєкту набір можливостей, серед яких:

* створити обліковий запис та за потреби видалити його
* увійти в обліковий запис та вийти з нього
* відредагувати власний обліковий запис (змінити ім’я, пароль, пошту, телефон або аватар)
* переглянути облікові записи інших користувачів
* приєднатися до проєкту та за потреби вийти з нього
* створити задачу та видалити її
* прикріпити до задачі файл
* редагувати задачу (змінити її статус, теги, дедлайн або виконавця)
* відфільтрувати задачі
* написати адміністратору за підтримкою


## Практичність

- Інтуїтивний та зрозумілий інтерфейс
- Можливість кастомізувати інтерфейс (змінити кольорову гаму, шрифт та його розмір)
- Кросплатформеність
- Підтримка декількох мов
- Наявність готових темплейтів для роботи із системою
- Можливість працювати офлайн

## Надійність

- Багаторівнева автентифікація
- Система доступу на основі ролей
- Фільтрація SQL injection
- Відмовостійкість
- Швидке відновлення після збоїв
- Резервування даних

## Продуктивність

- Швидка синхронізація змін, внесених декількома користувачами в один проєкт
- Ефективне використання пам'яті
- Кешування даних
- Розподілення і обмеження ресурсів для кожного проєкту (встановлення максимальної кількості учасників проєкту та запитів за одиницю часу)

## Експлуатаційна придатність

- Наявність зрозумілої документації
- Можливість звернутися до онлайн-підтримки
- Підтримка роботи різних версій системи
- Можливість масштабування системи за великого попиту
